    Implementation of the PEER NGA-East ground motion model. This is a custom version of the model developed specifically for USGS applications. It is a composite model that consists of a reduced set of 13 median ground motion models (down from 29 in the full NGA-East model) with period dependent weights. PEER recommends the use of a total, magnitude dependent ergodic sigma model. This implementation includes the 3-branch logic tree on sigma as well.

    Calculation of hazard using this preliminary implementation deviates somewhat from the current nshmp-haz PSHA pipeline and required implementation of a MultiScalarGroundMotion. A MultiScalarGroundMotion stores arrays of means and sigmas with associated weights and can only be properly processed by ExceedanceModel.NSHM_CEUS_MAX_INTENSITY at this time.

    This class also manages implementations of the 19 'seed' models used to generate (via Sammons mapping) the 13 NGA-East models and associated weights. Ground motions for the 19 seed and 13 component models are computed via table lookups.

    Note: Direct instantiation of GroundMotionModels is prohibited. Use Gmm.instance(Imt) to retrieve an instance for a desired Imt.

    Implementation note: Mean values are NOT clamped per GmmUtils.ceusMeanClip(Imt, double).

    Reference: Goulet, C., Bozorgnia, Y., Kuehn, N., Al Atik L., Youngs, R., Graves, R., Atkinson, G., 2017, NGA-East ground-motion models for the U.S. Geological Survey national seismic hazard maps: PEER Report No. 2017/03, 180 p.
